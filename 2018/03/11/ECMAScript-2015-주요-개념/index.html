ECMAScript 2015 주요 개념

Mar 11, 2018 in JavaScript

들어가며

ECMAScript 2015 (ES6) 를 학습하며, 주요 개념을 정리, 메모 한다. 앞으로 ES6 를 사용하는데 있어 꼭 기억해야 하는 부분만 기록하도록 한다.


let 과 this, 변수 Scope, 그리고 const

var

ES5 이하에서 변수 선언은 var 키워드를 사용한다. var 를 사용하지 않고도 변수를 사용할 수 있다. 이는 해당 변수가 글로벌 오브젝트(window) 에 작성됨을 의미한다. var 키워드를 이용하여 변수를 전역으로 선언하였을 경우도, 글로벌 오브젝트에 작성된다.

글로벌 오브젝트에 변수를 선언하는 것은 위험하다. 다른 어플리케이션에서 접근할 수 있기 때문이다. 페이지 내에 “use strict” 키워드를 추가하여, var 키워드 없이 변수를 선언하는 것을 막을 수 있다.

let

ES6 에서 추가되었으며, var 키워드의 문제점을 보완하였다. { } 단위로 Scope 를 지정할 수 있다. (중요)

  1. 함수안에 작성한 let 변수는 함수가 스코프이다.
  2. if( ){ let sports = “축구” } 의 경우 sports 의 Scope 는 { } 이다.
  3. let 변수는 호이스팅 되지 않는다.

let 과 this

console.log() 가 실행되는, 현재의 스코프는 글로벌 오브젝트(window) 이다. 따라서 this.music 은 음악이 출력되지만, this.sports 는 undefined 가 출력된다.

더 정확히..
자바스크립트에 글로벌 오브젝트는 있지만 window 오브젝트는 없다. window 오브젝트에는 DOM 뿐만 아니라, window 오브젝트 자체의 프로퍼티도 존재한다. window 오브젝트 내에 글로벌 오브젝트가 존재한다고 볼 수 있다. window 오브젝트는 브라우저의 자바스크립트 엔진이 생성한다.

function 도 별도의 Scope 를 가진다. Scope 내의 Scope 가 있는 구조로, 계층 구조이다. window.get() 과 같이 get() function 앞에 오브젝트의 위치를 지정하게 되면, function 내의 this 는 해당 오브젝트(window) 를 참조하게 된다.

let 과 호이스팅

var 변수는 호이스팅 되지만, let 변수는 호이스팅 되지 않는다.

const

const 키워드는 변수에 할당된 값을 변경할 수 없다.

const 키워드로 선언된 obj 에 새로운 오브젝트 {} 를 할당하는 것은 불가하나, obj 의 language 프로퍼티에 값을 변경하는 것은 가능하다.


arrow function

arrow function 은 function(param) {code} 형태를 축약한 것으로 (param) => {code} 형태로 작성한다. arrow function 은 익명(anonymous) 함수이다.

arrow function 은 { } 를 사용하지 않고, 한줄에 작성할 수 있다. 파라미터 수가 하나이면, ( ) 를 제외할 수 있다. 파라미터가 없으면 ( ) 만 작성한다.

arrow function 과 return

fn1 의 경우, { } 내에 return 이 없으므로 undefined 를 출력(ES5) 한다. JavaScript 는 ( ) 안의 코드를 표현식(expression) 으로 인식한다. 따라서 fn2 는 3 을 출력한다. fn3 의 경우, ( ) 내에 오브젝트({ })를 선언하였기 때문에 {} 를 출력한다. 만약, fn1 의 의도가 오브젝트 return 이라면, fn3 형태로 작성하여야 한다.

arrow function 과 new 연산자

arrow function 은 new 연산자로 인스턴스를 생성할 수 없다. new 연산자를 사용하기 위해서는, function 내에 prototype 과 constructor 가 연결되어 있어야 한다. arrow 형태로 선언된 function 은 prototype 이 연결되어 있지 않고, 따라서 new 연산자를 사용할 수 없다. (크롬 개발자 도구로 확인)

arrow function 과 arguments

function 키워드로 선언된 함수는, arguments 오브젝트에 function 호출 시, 전달된 파라미터가 저장된다. 하지만 arrow function 에는 arguments 가 존재하지 않는다.

arrow function 은 arguments 대신, rest 파라미터를 사용한다. let fn1 = (…rest) => {code} 와 같은 형태이며, function 호출 시 전달된 파라미터가 rest 에 배열 형태로 저장된다.

arrow function 과 this

new 연산자로 Sports 인스턴스를 생성하여, newSports 변수에 할당하였다. Sports 의 생성자 함수에서 this.count 에 20을 할당하였다. 이때 this 는 생성하는 인스턴스를 참조하므로 count 프로퍼티에 20이 설정된다.

newSports.get() 함수를 호출하면, get() 함수에 작성된 setTimeout() 함수가 실행되며 1초 후에 콜백 함수가 실행된다. setTimeout() 이 window 오브젝트의 함수이므로, this 는 window 오브젝트이다. 따라서 this.plus 는 window 오브젝트 내에 plus 함수가 없기 때문에 undefined 를 호출하게 된다.

setTimeout 의 콜백 함수를 arrow function 으로 작성하였다. 이 경우 콜백 함수 내의 this 는 newSports 인스턴스를 참조한다. 따라서 this.plus() 가 호출되어 this.count 의 값을 1 증가시키고, this.count 는 21을 출력하게 된다.

prototype 에 arrow function 을 연결하면, 함수 내의 this 는 window 오브젝트를 참조한다. 따라서 this.count += 1; 은 newSports 의 count 에 1을 더하지 못하고, this.count 는 20을 출력한다.

arrow function 의 this 는, 실행 환경 관점에서, 상위 오브젝트의 this 를 바인딩하는 것으로 생각된다. (주관적인 이해)
